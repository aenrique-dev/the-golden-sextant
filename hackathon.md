## 🟧 PROJECT IDEAS (Suggested by ELT)

### 1. ENGINEERING

#### 1.1 Engineering Q&A Agent
Build an AI agent that helps answer technical questions typically directed to Engineering.

**Current pain point:** TAMs and SEs frequently need technical details that only developers can provide, which slows response times and takes too much time from engineers.

**Possible solution:**
1. Create an AI agent connected to our codebase
2. The agent answers technical questions automatically
3. Engineering can review and correct responses when needed, improving the system over time

#### 1.2 Living Production Environment Map
An authoritative, live map of what production systems exist, what they do, what data they handle, how they're built, how they're connected, and when they change from new commits.

Enabled use cases:
- Visualization layer for onboarding and decision-making
- Data source for a chatbot
- Better connections to security monitoring and uptime tools
- Feeds to systems like Drata (audit evidence automation)
- Inventory of open source and encryption libraries used

Supports **Corporate Priority #5: Ensuring Product Security Practices Protect Customer Data**. Please interview Samir/Alfredo and Paul about their requirements.

### 2. FINANCE

#### 2.1 Accounting
Agents to run workflows for prepaid and accruals for the month-close.

#### 2.2 Operations/Data
Agents to create workflows that keep module-per-product-per-customer live — pull data from Postgres and SFDC and feed to a live dashboard.

### 3. CS / TAM / SUPPORT

#### 3.1 Value and Outcomes Agent
Connects to SFDC to extract the 3 Whys and Objectives/use cases for a customer. Maps data from the customer tenant to those objectives and constructs a value proposition readout for TAMs and AEs to use in stakeholder and EB conversations.

#### 3.2 Engagement Quality Agent
Analyzes calls, meetings, and notes across sources to score how engaged an account is and whether value is being acknowledged. Generates a scorecard based on engagement level, value communicated, and customer acknowledgement.

#### 3.3 Progress Over Time Agent
Retrieves data from customer environments periodically to extract what areas of the platform a customer is making progress in — compliance outcomes, integration reduction, threat detection trends, etc.

### 4. REVENUE MARKETING

[Detailed request link](https://docs.google.com/document/d/11ud0kmfuJK3In9yMLL55MVN_lFeSo8AlDXl5gj_pcIE/edit?usp=sharing)

#### 4.1 Account ICP Normalization Agent
Takes raw CRM contact data (job titles + account info) and outputs a clean, structured view of key stakeholders at each account, normalized by persona and organized into likely buying committees.

#### 4.2 Content Review Agent
An AI-powered pre-flight review tool for marketing content. Runs drafts through 5 structured checks before PMM review, returning a scoped, prioritized list of issues — separating "fix this yourself" from "flag for PMM."

#### 4.3 Competitor Monitoring Agent
Surfaces new blogs, product updates, webinars, events, use cases, partners, and customers from a provided competitor list by scraping their web pages daily.

#### 4.4 (Bonus) Event Logistics Agent
Automates Field Marketing event setup by collecting event details once and creating required assets across SFDC, Google Sheets, Calendar, Slack, Asana, and landing pages.
